Enact 04 / 12

The Problem

The tooling layer is broken

⚠

Protocols are fragmenting

MCP, CLI tools, API wrappers — new interfaces keep emerging. MCP has context bloat issues, CLI tools lack discovery, and nothing has converged. The protocol is a moving target.

✕

No trust model

"Install this tool" means running unaudited code with host access. No signing, no verification, no sandboxing. Enterprises won't accept this.

⚠

No governance or visibility

Which agents use which tools? Who approved this? When something breaks, teams dig through logs across systems. No audit trail, no access control, no observability.

Every approach has the same gap: no standard way to package, distribute, trust, or govern agent tools. That layer is missing regardless of how agents talk to tools.

The Product

Understand

04

Manage

Pin versions, set policies, control access by team across every tool type

Every invocation flows through Enact. That's a natural instrumentation point: which tools agents use, how often, success/failure rates, latency, cost per team. Observability built into the architecture.
