The Solution

A universal registry for agent skills

Enact is npm for AI tools. Agents discover capabilities at runtime through semantic search, verify trust through cryptographic signing, and execute skills in portable containers ‚Äî all through a single protocol.

# Find a tool
enact search scraper

# Read its documentation
enact learn enact/firecrawl

# Run it
enact run enact/firecrawl:scrape -a '{"url": "https://example.com"}'

How It Works

Search ‚Üí Learn ‚Üí Run

01

Discover

Agent searches the registry using natural language. Vector embeddings return results by capability, not just keywords.

02

Learn

Agent reads the SKILL.md ‚Äî agent-facing documentation that teaches it how to invoke the tool, what parameters it needs, and what to expect.

03

Verify

Sigstore-based cryptographic verification confirms publisher identity, package integrity, and transparency log entry.

04

Execute

Skills run in the configured backend ‚Äî local, Docker, Dagger, or remote ‚Äî with secrets injected at runtime, never exposed.

Why Enact

Built different

Security

Cryptographic Trust

Every package is Sigstore-signed. Publisher identity, integrity, and transparency are verified before execution. Not an afterthought ‚Äî foundational.

Discovery

Semantic Search

Vector embeddings index every skill. Agents find tools by describing what they need, not by memorizing package names.

Execution

Portable Runtime

Skills declare dependencies. You choose where they run ‚Äî local, Docker, Dagger, or remote. Same skill, any backend, zero changes.

Ownership

Self-Hosted Registry

One command to run your own private registry. SQLite + local storage. No vendor lock-in. Mirror the public registry or run fully private.
